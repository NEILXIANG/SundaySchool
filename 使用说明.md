# 主日学照片整理工具 - 使用说明（单一口径）

本仓库同时包含：
- **老师使用的打包版（推荐）**：双击运行，无需安装 Python
- **开发者源码版**：用于开发/测试/打包

为避免“入口与文档口径冲突”，这里不再重复写两套流程细节；请按身份阅读对应文档：

## 👩‍🏫 老师（打包版）
- 中文老师指南：[sunday-photos/doc/TeacherGuide.md](sunday-photos/doc/TeacherGuide.md)
- English guide: [sunday-photos/doc/TeacherGuide_en.md](sunday-photos/doc/TeacherGuide_en.md)

老师只需要记住：运行 `SundayPhotoOrganizer` 后，照片放到**可执行文件同目录**自动创建的 `input/` 里（`input/student_photos/` 与 `input/class_photos/`），再运行一次即可；输出会自动打开（`output/`）。

补充（macOS 首次运行建议）：如果你拿到的是“控制台发布包”，首次推荐直接双击 `启动工具.command` 作为入口（更符合 macOS 双击习惯）。

### 快速示例（老师最常见的目录结构）

解压后的文件夹通常长这样（你的文件名可以不同）：

```
SundayPhotoOrganizer/
├── 启动工具.command / 启动工具.sh / SundayPhotoOrganizer.app / Launch_SundayPhotoOrganizer.bat
├── input/
│   ├── student_photos/
│   │   ├── 张三/张三_1.jpg
│   │   └── 李四/李四_1.jpg
│   └── class_photos/
│       ├── YYYY-MM-DD_活动_001.jpg
│       └── YYYY-MM-DD_活动_002.jpg
├── output/
└── logs/
```

### FAQ（只放最常见的 4 个）

- **macOS 提示“无法打开/无法验证开发者”**：按提示去“系统设置 → 隐私与安全性 → 仍要打开”。
- **双击后一闪就没了/没反应**：通常是照片没放到 `input/student_photos/` 或 `input/class_photos/`，或点错入口。
- **运行后没有 output**：先看 `logs/`，再看 `output/*_整理报告.txt` 是否生成（有时 output 为空但报告会提示原因）。
- **需要求助怎么发**：把整个 `logs/` 文件夹打包 + 一张报错截图（如果有）。
