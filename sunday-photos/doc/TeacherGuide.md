# 老师使用指南（打包版 / 不需要安装 Python）

适用对象：直接双击运行发布包的老师。

本指南以“最少操作、最少疑惑”为目标：老师只需要记住一件事——
**把照片放进桌面自动生成的文件夹，然后再双击运行一次即可。**

## 运行方式（macOS）
- 双击 `release_console/启动工具.sh`（推荐）。
- 或双击 `release_console/SundayPhotoOrganizer`。

> 若 macOS 第一次运行被阻止：系统设置 → 隐私与安全性 → 仍要打开。

## 第一次运行会发生什么
- 程序会在桌面创建文件夹：`主日学照片整理/`
- 并自动生成这些子文件夹：
  - `student_photos/`：学生参考照
  - `class_photos/`：课堂照片
  - `output/`：整理结果
  - `logs/`：日志（遇到问题时用）

## 放照片（老师只需要做这一步）
- 学生参考照：放到 `桌面/主日学照片整理/student_photos/`
  - 命名：`姓名.jpg` 或 `姓名_序号.jpg`（如：张三.jpg、张三_2.jpg）
- 课堂照片：放到 `桌面/主日学照片整理/class_photos/`
  - 建议按日期建子目录：`2025-12-21/照片.jpg`

## 再次运行
- 再双击运行一次，程序会开始识别与分类。
- 整理完成后，会自动打开 `output/`。

## 重要提示（避免“惊吓点”）
- 程序可能会把 `class_photos` 根目录的照片**按日期移动**到 `YYYY-MM-DD/` 子目录中，这是正常现象（为了增量处理与便于查找）。

## 调整识别准确度（可选）
- 桌面目录下有 `config.json`，可修改：`face_recognition.tolerance`
  - 默认 `0.6`；数值越低越严格（更少误认，但可能漏认）
  - 建议范围：`0.45 ~ 0.75`

## 常见问题
- 程序很快结束：通常是 `student_photos/` 或 `class_photos/` 里没照片，按提示放好后再运行。
- 识别不准：给该学生补 2-3 张清晰正脸参考照；或把 `tolerance` 调低一点（如 0.55 / 0.5）。
- 需要求助：请把 `桌面/主日学照片整理/logs/` 里最新的日志文件发给技术支持。
